# S3 / MinIO

Store repository data in any S3-compatible object storage service: Amazon S3, MinIO, Backblaze B2, Wasabi, Cloudflare R2, or any other S3-compatible endpoint.

---

## URL Format

```
s3:bucket/prefix
s3:region/bucket/prefix
```

---

## Configuration

```yaml
repositories:
  s3-prod:
    url: s3:my-bucket/squirrel/prod
    password: ${env:SQUIRREL_PASSWORD}
    env:
      AWS_ACCESS_KEY_ID: ${env:AWS_ACCESS_KEY_ID}
      AWS_SECRET_ACCESS_KEY: ${env:AWS_SECRET_ACCESS_KEY}
      AWS_DEFAULT_REGION: eu-central-1
```

---

## Environment Variables

| Variable | Description |
|---|---|
| `AWS_ACCESS_KEY_ID` | S3 access key ID |
| `AWS_SECRET_ACCESS_KEY` | S3 secret access key |
| `AWS_DEFAULT_REGION` | AWS region (default: `us-east-1`) |
| `AWS_ENDPOINT_URL` | Custom endpoint for MinIO or S3-compatible services |
| `AWS_SESSION_TOKEN` | Session token for temporary credentials |

These can be set globally or scoped per-repository using the `env:` field.

## AWS S3

```yaml
repositories:
  s3-prod:
    url: s3:my-bucket/squirrel
    password: ${env:SQUIRREL_PASSWORD}
    env:
      AWS_ACCESS_KEY_ID: ${env:AWS_ACCESS_KEY_ID}
      AWS_SECRET_ACCESS_KEY: ${env:AWS_SECRET_ACCESS_KEY}
      AWS_DEFAULT_REGION: eu-central-1
```

### With IAM Role (EC2 / ECS)

When running on EC2 or ECS with an IAM role, credentials are retrieved automatically from the instance metadata service. No access key variables are needed:

```yaml
repositories:
  s3-prod:
    url: s3:my-bucket/squirrel
    password: ${env:SQUIRREL_PASSWORD}
    env:
      AWS_DEFAULT_REGION: eu-central-1
```

## MinIO

```yaml
repositories:
  minio:
    url: s3:squirrel-backups/prod
    password: ${env:SQUIRREL_PASSWORD}
    env:
      AWS_ACCESS_KEY_ID: minioadmin
      AWS_SECRET_ACCESS_KEY: ${env:MINIO_SECRET}
      AWS_ENDPOINT_URL: http://minio.example.com:9000
```

For HTTPS MinIO:

```yaml
      AWS_ENDPOINT_URL: https://minio.example.com
```

## Backblaze B2 (S3-compatible)

```yaml
repositories:
  b2:
    url: s3:my-b2-bucket/squirrel
    password: ${env:SQUIRREL_PASSWORD}
    env:
      AWS_ACCESS_KEY_ID: ${env:B2_APPLICATION_KEY_ID}
      AWS_SECRET_ACCESS_KEY: ${env:B2_APPLICATION_KEY}
      AWS_ENDPOINT_URL: https://s3.us-west-004.backblazeb2.com
      AWS_DEFAULT_REGION: us-west-004
```

---

## Notes

- Bucket must exist before running `squirrel init`.
- Enable bucket versioning on AWS S3 for extra protection against accidental deletion.
- squirrel uses multipart uploads for large pack files.
- Server-side encryption (SSE-S3 or SSE-KMS) can be enabled on the bucket independently of squirrel's own encryption.
